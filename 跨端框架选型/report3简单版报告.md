

# circuit 移动端技术选型（简版）

## 1. 背景

circuit 是 AI 问答平台（RAG + 多模态），目前已有 **Web App**，需要支持 **iOS 和 Android**。

移动端技术主要有三类：

- **Web App**：浏览器运行
- **跨端**：一套代码运行 iOS + Android
- **Native**：iOS / Android 原生开发

由于 **Web App 已存在**，主要需要在 **跨端 vs Native** 之间选择。

------

## 2. 三类方案整体对比

| 维度     | Web App | 跨端  | Native |
| -------- | ------- | ----- | ------ |
| 性能     | ★★☆☆☆   | ★★★★☆ | ★★★★★  |
| 开发效率 | ★★★★★   | ★★★★☆ | ★★☆☆☆  |
| 成本     | ★★★★★   | ★★★★☆ | ★★☆☆☆  |
| 招聘     | ★★★★★   | ★★★★☆ | ★★☆☆☆  |

结论：

- **Web App**：开发最快，但性能和体验上限最低
- **Native**：性能最好，但成本最高
- **跨端**：性能接近原生，同时效率和成本更可控

对 **AI 问答类应用**来说，跨端性能通常已经足够。

------

## 3. 跨端方案对比

当前主流跨端方案：

**React Native / Flutter / Ionic / Kotlin Multiplatform**

| 方案                 | 性能  | 生态  | 招聘  | 维护  |
| -------------------- | ----- | ----- | ----- | ----- |
| React Native         | ★★★★☆ | ★★★★★ | ★★★★★ | ★★★★☆ |
| Flutter              | ★★★★★ | ★★★★☆ | ★★★☆☆ | ★★★☆☆ |
| Ionic                | ★★☆☆☆ | ★★★☆☆ | ★★★★★ | ★★★★★ |
| Kotlin Multiplatform | ★★★★☆ | ★★☆☆☆ | ★★☆☆☆ | ★★☆☆☆ |

结论排序：

**React Native > Flutter > Ionic > Kotlin Multiplatform**

简要判断：

- **React Native**：综合最均衡，前端团队友好，招聘池最大
- **Flutter**：性能很好，但需要 Dart 体系
- **Ionic**：本质 WebView，性能和体验上限较低
- **Kotlin Multiplatform**：更偏原生团队方案

------

## 4. Native 方案参考

Native 主要技术：

| 语言              | 性能  | 生态  | 招聘  | 维护  |
| ----------------- | ----- | ----- | ----- | ----- |
| Swift（iOS）      | ★★★★★ | ★★★★☆ | ★★★★☆ | ★★★★☆ |
| Kotlin（Android） | ★★★★★ | ★★★★★ | ★★★★★ | ★★★★☆ |

原生优势：

- 系统能力最完整
- UI 与系统体验完全一致
- 性能上限最高

缺点是 **开发成本和人力投入明显更高**。

------

## 5. 推荐路线

**建议：跨端为主 + 原生兜底**

原因：

- 跨端负责大部分业务页面，开发效率更高
- 原生负责系统能力和关键性能链路

落地架构：

**React Native + Swift / Kotlin**

RN 负责主业务，原生负责底层能力。
